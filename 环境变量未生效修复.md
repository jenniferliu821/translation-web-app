# 🔧 Railway 环境变量未生效 - 修复步骤

## 当前状态
- ✅ 网站能打开
- ✅ Variables 里有 GOOGLE_API_KEY
- ❌ 应用没有读取到环境变量（日志显示警告）

---

## 🚀 最快修复方法

### 步骤 1：删除现有的环境变量
1. 在 Railway 中，点击您的服务 `translation-web-app`
2. 点击 **Variables** 标签
3. 找到 GOOGLE_API_KEY
4. 点击右边的 **删除图标** 🗑️ 或 "×"
5. 确认删除
6. **等待 10 秒**

### 步骤 2：重新添加环境变量
1. 在同一个 Variables 页面
2. 点击 "**New Variable**" 或 "**+ Add Variable**"
3. 输入（仔细检查，确保没有多余空格）：
   ```
   Name: GOOGLE_API_KEY
   Value: [粘贴您的完整新 API Key]
   ```
4. 点击 "**Add**" 保存

### 步骤 3：强制完全重启
这一步最关键！必须完全重启才能让变量生效：

**方法 A：重启服务（推荐）**
1. 点击 Settings 标签
2. 向下滚动找到 "**Restart**" 按钮
3. 点击 Restart
4. 等待重启完成（30秒-1分钟）

**方法 B：重新部署**
1. 点击 Deployments 标签
2. 点击右上角 "**⋯**" (三个点)
3. 选择 "**Redeploy**"
4. 等待部署完成（2-3分钟）

### 步骤 4：验证
1. 部署完成后，再次查看日志：
   - Deployments → 点击最新部署 → Deploy Logs
2. 应该看到：
   ```
   🔧 配置 Gemini API...
      API Key 长度: 39              ← 成功！
      API Key 前缀: AIzaSy...
   ✅ Gemini 配置完成
   ```

如果还是看到警告，继续往下看其他方法。

## 🎯 为什么会这样？

Railway 的环境变量可能遇到这些问题：
1. **缓存问题** - 旧容器缓存了空的环境变量
2. **作用域错误** - 变量在项目级而非服务级
3. **重启不彻底** - Redeploy 有时不会完全重新加载环境变量

**完全删除并重建服务是最彻底的解决方案。**
