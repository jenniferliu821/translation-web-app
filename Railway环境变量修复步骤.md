# 🔧 Railway 环境变量未生效 - 详细修复步骤

## 当前问题
Railway 部署失败，报错：`ValueError: ❌ 错误：未设置 GOOGLE_API_KEY 环境变量！`

即使您已经在 Railway Variables 中添加了新的 API Key，但是没有生效。

---

## 🎯 立即修复方案（按顺序执行）

### 方法 1：正确保存环境变量 ⭐ 最可能解决

#### 步骤 A：删除旧变量
1. 在 Railway 项目中，点击 **Variables** 标签
2. 找到现有的 `GOOGLE_API_KEY` 变量
3. 点击变量右侧的 **垃圾桶图标** 🗑️ 删除它
4. 确认删除

#### 步骤 B：重新创建变量
1. 点击 **New Variable** 或 **+ Variable** 按钮
2. 在 **Variable Name** 输入：`GOOGLE_API_KEY`（注意全部大写，无空格）
3. 在 **Value** 输入您的新 API Key
4. **关键**：点击 **Add** 按钮保存

#### 步骤 C：手动触发重新部署
1. 回到 **Deployments** 标签
2. 点击右上角的 **"⋯"** (三个点) 菜单
3. 选择 **"Redeploy"** 或 **"Trigger Deploy"**
4. 等待 2-3 分钟观察部署状态

---

### 方法 2：检查变量名称是否正确

**常见错误：**
- ❌ `Google_API_KEY`（大小写错误）
- ❌ `GOOGLE_API_KEY `（末尾有空格）
- ❌ `GOOGLE API KEY`（有空格）
- ✅ `GOOGLE_API_KEY`（正确）

**检查步骤：**
1. 在 Variables 标签中
2. 仔细查看变量名称
3. 必须完全匹配：`GOOGLE_API_KEY`
4. 如果不对，删除重建

---

### 方法 3：检查您的 Railway 项目结构

Railway 可能有多个 Service，确保添加变量到正确的 Service：

1. 在项目概览页面，查看有几个服务卡片
2. 点击 **translation-web-app** 服务（不是其他服务）
3. 在这个服务内点击 **Variables** 标签
4. 确认 `GOOGLE_API_KEY` 在这个服务下

---

### 方法 4：使用 Railway CLI 添加（备用方案）

如果 Web 界面有问题，可以用命令行：

```bash
# 安装 Railway CLI（如果还没安装）
npm install -g @railway/cli

# 登录
railway login

# 链接项目
railway link

# 添加环境变量
railway variables set GOOGLE_API_KEY="您的新API-Key"

# 触发重新部署
railway up
```

## 💡 常见 Railway 环境变量问题

### 问题 1：编辑而不是删除重建
- **现象**：点击编辑修改值后，变量不生效
- **解决**：必须删除旧变量，重新创建

### 问题 2：忘记手动重新部署
- **现象**：添加变量后，Railway 没有自动重启
- **解决**：手动点击 Redeploy

### 问题 3：变量添加到错误的环境
- **现象**：在 Production 环境添加，但实际部署在 Development
- **解决**：确认当前活跃环境
