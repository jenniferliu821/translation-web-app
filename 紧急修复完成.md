- Railway 项目页面 → Variables 标签

#### 3. 检查环境变量的应用范围
Railway 的环境变量有两种类型：
- **Shared Variables** - 整个项目共享
- **Service Variables** - 特定服务专用

**重要：确保 GOOGLE_API_KEY 被添加到了正确的服务！**

## 📋 详细排查步骤

### 方法 1：重新添加环境变量（推荐） ⭐

1. **完全删除现有变量**
   - 在 Variables 标签中
   - 找到 GOOGLE_API_KEY
   - 点击垃圾桶图标 🗑️
   - 确认删除
   - **等待 10 秒**

2. **重新创建变量**
   - 点击 "New Variable" 或 "+ Add"
   - 输入：
     ```
     Key: GOOGLE_API_KEY
     Value: [粘贴您的新 API Key]
     ```
   - **重要：确保没有多余的空格！**
   - 点击 Add

3. **手动触发重新部署**
   - Deployments 标签
   - 点击右上角 "⋯" → "Redeploy"
   - 等待部署完成

### 方法 2：检查变量作用域

有时环境变量被添加到了错误的位置：

1. **项目级别变量**
   - 点击项目名称（顶部）
   - 查看是否有 Variables 选项

2. **服务级别变量**
   - 点击您的服务卡片（translation-web-app）
   - 查看 Variables 标签

**确保在服务级别添加了 GOOGLE_API_KEY！**

### 方法 3：通过 Railway CLI（如果上述方法都失败）

如果 Web 界面有问题，可以用命令行：

```bash
# 安装 Railway CLI
brew install railway

# 登录
railway login

# 链接项目
cd ~/Desktop/cursor/多语言翻译
railway link

# 设置环境变量
railway variables set GOOGLE_API_KEY="你的新API密钥"

# 重新部署
railway up
```

## ❓ 常见问题

### Q1: 我确定添加了 GOOGLE_API_KEY，为什么还是不工作？

**可能原因：**
1. 变量值有多余的空格或换行
2. 添加到了错误的服务
3. Railway 缓存了旧的环境变量

**解决：**
- 完全删除变量，等 10 秒，重新添加
- 手动触发 Redeploy
- 检查是否添加到了正确的服务

### Q2: 日志显示 API Key 长度为 0 或为空

**说明环境变量没有被读取到！**

**解决：**
- 检查变量名拼写：必须是 `GOOGLE_API_KEY`（全大写）
- 检查是否在服务级别（不是项目级别）
- 尝试使用 Railway CLI 设置

### Q3: 新的 Google API Key 是否有效？

**测试新 API Key：**
1. 在本地测试：
   ```bash
   cd ~/Desktop/cursor/多语言翻译
   export GOOGLE_API_KEY="你的新密钥"
   python test_api.py
   ```

2. 应该看到：
   ```
   ✅ API 配置成功
   ✅ 翻译测试成功
   ```
